# SaaS copy edit — fill-in template

Use this template when humanising SaaS feature copy, pricing copy, or
in-product explainers. Keep feature names, plan names, and pricing exact.

---

## Inputs

- **Section type:** feature block / pricing / in-product explainer /
  onboarding / changelog / [other]
- **Product name:** [exact, must be preserved]
- **Plan names:** [exact, if rewriting pricing]
- **Audience:** [end user / admin / billing owner / developer]
- **Must-keep items:** feature names, plan names, prices, currencies,
  limits, SEO keywords, CTAs.
- **Word limit per block:** [optional]

---

## Mode + assumptions

> Mode: SaaS feature copy · Tone: clear, specific, outcome-led
> Assumption: [one line — only if the source had unclear intent]

---

## Rewritten content

````markdown
### [Feature name — kept exact]

[1–2 sentences. What it does. Who it is for. What changes after.
No em dashes. No "robust / seamless / empower / leverage / game-changing".]

- [What it actually does — concrete action]
- [Who uses it — named role or audience]
- [What changes after — specific outcome]

[Optional: short example. "E.g. for a dental clinic, this sends WhatsApp
reminders 24 hours before each appointment."]

---

### [Next feature…]

[…]
````

For pricing copy:

````markdown
### [Plan name — kept exact] — [Price — kept exact]

[One short line: who this plan is for.]

- [Limit — exact number from source]
- [Limit — exact number from source]
- [Included support / SLA — exact from source]

[CTA — kept from source]
````

## Change note

- Replaced banned hype words: [list, e.g. "robust → reliable",
  "leverage → use"].
- Replaced vague benefits with concrete outcomes: [list].
- Preserved feature names, plan names, prices, and limits exactly.
